# Release history

Per-release retros for paper-wiki — the "why" behind each tag, separated
from the day-to-day engineering plan and from the user-facing changelog.

- **`CHANGELOG.md`** at the repo root is the canonical, audience-facing
  list of what shipped each release (Keep a Changelog format).
- **`tasks/plan.md`** is the working engineering plan; it is gitignored
  and lives only in the maintainer's clone.
- **`docs/release-history/vN.M.P.md`** (this directory) is the tracked
  retro per release: short Background + bulleted "What shipped" +
  "Lessons learned" + cross-references to the commit / tag / CHANGELOG
  entry. Retros are written at the release author's discretion — most
  releases are well-served by the CHANGELOG alone, so a retro is added
  only when the release is otherwise hard to reconstruct after the fact
  (an unusual decision was taken, a near-miss happened, or the release
  is a single-task feature whose context lives mostly in the commit
  body).

For end-user release notes, see [`CHANGELOG.md`](../../CHANGELOG.md).
